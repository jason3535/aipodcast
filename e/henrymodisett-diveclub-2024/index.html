本期速览 · Overview

Perplexity 设计负责人 Henry Modisett 分享了他如何在不知道产品是什么的情况下构建组件库，以及一个粗糙的原型如何成为不可或缺的工具。

Henry Modisett, design lead at Perplexity, shares how he built a component library before knowing the product, and how a scrappy prototype became an indispensable tool.

要点 · TL;DR

  • 设计师应会编码以建立同理心并更快交付。
    Designers should code to build empathy and ship faster.
  • 消费产品必须快速、熟悉且零学习成本。
    Consumer products must be fast, familiar, and require zero learning.
  • 产品应更强大而不显复杂。
    Products should get more powerful without feeling more complex.

核心观点 · Key points

  • 设计师应该会编码，以建立同理心并更快交付。
    Designers should code to build empathy and ship faster.
  • 消费产品必须快速、熟悉且无需学习。
    Consumer products must be fast, familiar, and require zero learning.
  • 初创公司的品牌策略应优先考虑招聘和差异化。
    Brand strategy for startups should prioritize recruiting and differentiation.
  • 不惜一切代价追求速度需要清晰的决策和赋能设计师。
    Velocity at all costs requires clear decision-making and empowering designers.
  • 产品应变得更强大，但不应感觉更复杂。
    Products should get more powerful without feeling more complex.

反共识 · Contrarian takes

  • 设计系统是语义上的；代码复用至关重要，而非 Figma 组件库。
    Design systems are semantic; code reuse is essential, not Figma libraries.
  • 并非每个细节都需要完美；在值得做好的地方追求投资回报率。
    Not every surface needs perfection; chase ROI on what to make great.
  • 品牌指南应成为设计师表达自我的动态平台。
    Brand guidelines should be dynamic platforms for designer expression.
  • AI 硬件将像收音机一样在设计上竞争，技术商品化。
    AI hardware will compete on design like radios did, commoditizing tech.
  • 不要在交互设计上创新；使用熟悉的模式以加快速度。
    Don't innovate on interaction design; use familiar patterns for speed.
